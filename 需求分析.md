## 4. 需求分析

### 4.1 用户需求

#### 4.1.1 小学生需求
- 希望题目像游戏一样有趣，做题过程不枯燥，有闯关感和成就感
- 题目字体要大、颜色鲜明，页面不能太复杂，一眼就能看懂怎么操作
- 答对时希望看到鼓励反馈（星星、金币、动画），答错时不要直接打击，最好有提示再试一次
- 希望有角色或形象陪伴（如小动物、小老师），增强代入感
- 做题时间不能太长，一次 5-10 分钟即可，避免疲劳
- 希望能看到自己过了多少关、得了多少星，有一种"变强了"的直观感受

#### 4.1.2 家长需求
- 能看到孩子每天/每周的练习记录和学习进度，而不是盲目地让孩子"做点题"
- 希望有正确率、常错题型、用时变化等数据，便于判断孩子在哪个知识点薄弱
- 能设置每日练习题量或练习时长上限，防止孩子过度使用
- 希望系统提供错题回顾功能，让孩子有针对性地补弱
- 希望能收到简单的学习周报或提醒，知道孩子是否坚持练习
- 不希望在应用内出现社交、广告等干扰内容

#### 4.1.3 教师需求
- 能按年级或知识点快速筛选题目，生成一份课堂小练习或课后作业
- 希望题目难度精准匹配对应年级的教学大纲，不超纲、不偏门
- 能看到全班学生的完成情况和整体正确率分布，快速发现共性问题
- 支持导出或打印题目列表，方便在没有电子设备的情况下使用
- 布置练习的操作路径要短：选年级 → 选知识点 → 生成 → 分享/布置，三步以内完成

### 4.2 功能需求

#### 4.2.1 核心做题功能
- **关卡闯关**：按知识点和难度分章节、分关卡，玩家从第一关开始依次挑战，通关后解锁下一关
- **自动出题**：系统根据当前关卡难度和题型规则，自动生成数学题，保证每次练习的题目不完全相同
- **即时反馈**：每答完一道题，立刻显示对/错结果和正确答案，对时给正向反馈（星星+1、笑脸动画），错时给柔和提示（"再想想哦～"）
- **通关结算**：整关完成后弹出通关面板，显示得分、正确率、用时、获得星级，以及是否解锁新关卡
- **错题本**：答错的题目自动收录到错题本，按知识点分类，支持随时回看和重新练习
- **学习统计**：记录累计答题数、正确率趋势、各知识点掌握情况、练习时长等，以图表展示

#### 4.2.2 辅助功能
- **每日任务**：每天设置一个小目标（如"今天完成 10 道加法题"），完成后获得额外奖励
- **限时挑战**：部分关卡提供计时模式，在规定时间内完成可获得额外加分
- **提示功能**：遇到困难的题目，可消耗"提示次数"查看解题思路（不直接给答案）
- **进度云同步**（P2 第二期）：登录账号后，学习进度、错题本、成就等数据可跨设备同步
- **音效开关**：提供音效和背景音乐的开关，家长可控制
- **护眼模式**：支持护眼模式（暖色调），定时提醒休息

#### 4.2.3 家长/教师功能
- **家长控制台**：设置每日题量上限、练习时间段、年级选择
- **学习报告**：按周/月生成学习报告，包含正确率变化、薄弱知识点、练习时长统计
- **题目预览**：教师可在生成题目后预览全部题目，手动剔除或替换不合适的题
- **班级管理**（教师端）：创建班级、添加学生、批量查看练习数据

### 4.3 业务需求

#### 4.3.1 难度递进规则
- 难度递进必须严格遵循小学数学认知发展阶段：
  - **阶段一（5-6岁/一年级上）**：10以内加减法，题目以直观图形辅助理解
  - **阶段二（6-7岁/一年级下）**：20以内加减法（含进位退位），逐步减少图形辅助
  - **阶段三（7-8岁/二年级）**：100以内加减法、乘法口诀（1-9）
  - **阶段四（8-9岁/三年级）**：混合运算（加减乘除两步）、简单分数认识
  - **阶段五（9-10岁/四年级）**：多位数乘除法、小数加减、简单应用题
  - **阶段六（10-12岁/五-六年级）**：分数运算、综合应用题、图形面积体积计算
- 每个阶段对应一个章节，每章节下分 4-5 个小关卡，难度微调递进
- 某一关正确率 ≥ 80% 方可解锁下一关

#### 4.3.2 题目质量规则
- 每道题只考察一个知识点，不出现复合考点
- 题干文字不得超过 30 个字（低年级）或 60 个字（高年级）
- 选项的干扰项要有合理来源（如常见错误结果），不能随意编造
- 运算题结果必须为非负整数（低年级）或正有理数（高年级）
- 应用题场景取材于校园、家庭、购物、运动等小学生熟悉的生活场景
- 所有题目必须经过"小学生可理解度"审核，用词不得超出对应年级语文课本范围

#### 4.3.3 激励体系规则
- 正向激励为主，不做惩罚性扣分，避免产生畏难情绪
- 奖励分为三级：即时奖励（单题答对）→ 关卡奖励（通关）→ 成就奖励（里程碑）
- 连续答对 3 题触发"连击"状态，额外+2 分/题
- 连续答对 10 题获得"全连"徽章
- 每通过一个大章节获得一枚主题徽章
- 错题重做后答对，给予"翻盘奖励"（比首次答对多+1 分）

#### 4.3.4 错题处理规则
- 答错的题立即加入错题本，标注错题时间和所属知识点
- 同一知识点累计错题 ≥ 3 道时，系统主动推荐该知识点的专项练习
- 错题重做时先展示解题思路提示，再让用户作答
- 同一道错题连续重做正确 2 次后，从"活跃错题"中移除（但仍保留在历史错题中）
- 错题本支持按知识点、时间范围筛选

### 4.4 非功能需求

#### 4.4.1 性能要求
- 题目生成接口响应时间 ≤ 300ms，保证答题过程不卡顿
- 答题结果判定接口响应时间 ≤ 100ms，提交后立刻看到反馈
- 页面首次加载时间 ≤ 2 秒
- 支持至少 500 个学生同时在线答题，系统不降级
- 统计数据查询（周报、月报）响应时间 ≤ 2 秒

#### 4.4.2 可用性要求
- 界面按钮和文字大小适合小学生（建议按钮最小触控区域 44×44px）
- 色彩对比度符合 WCAG AA 级标准，确保视觉清晰
- 所有操作不超过 3 步即可触达核心功能
- 支持键盘和触屏两种操作方式
- 提供新手指引，首次进入时以动画方式引导完成第一道题
- 关键操作（如退出关卡、清空错题本）提供二次确认

#### 4.4.3 安全要求
- 用户密码加密存储（使用 bcrypt 或等效方案）
- 学习数据每日自动备份，防止丢失
- 接口防刷：同一用户 1 分钟内答题请求不超过 30 次
- 家长控制台需要密码或生物识别验证，防止孩子绕过设置
- 不收集任何超出必要范围的个人信息（仅需昵称、年级、密码）

#### 4.4.4 兼容性要求
- 支持 Chrome、Edge、Safari 最新两个大版本
- 支持 iPad 和主流 Android 平板（屏幕宽度 ≥ 768px）
- 后续优先适配微信小程序环境
- 支持离线做题（P3 远期规划：题目预加载到本地，网络恢复后同步数据）

#### 4.4.5 可维护性要求
- 题库和出题规则独立配置，方便后期调整难度和新增题型
- 前后端分离，接口文档使用 OpenAPI 3.0 规范
- 关键业务流程（出题、判题、统计）有单元测试覆盖
- 日志记录完整的答题流程，便于排查问题和分析学习行为

### 4.5 约束条件

#### 4.5.1 内容约束
- 题目知识点范围严格限定在《义务教育数学课程标准（2022 年版）》小学阶段内容之内
- 不出现负数运算（小学阶段不涉及）、不出现方程求解（初中内容）、不出现超出二年级生字表的文字
- 应用题的场景和物品选择必须积极健康，不涉及暴力、攀比、浪费等不良导向
- 不使用任何涉及宗教、政治、商业广告的内容

#### 4.5.2 技术约束
- MVP 阶段仅支持 Web 端（PC 浏览器 + 平板浏览器）
- 后端必须使用 Java 17 + Spring Boot
- 数据库使用 MySQL 8.0+
- 缓存使用 Redis 7.0+
- 前端框架在 Vue 3 和 React 18 之间二选一，MVP 阶段不做多端兼容

#### 4.5.3 资源约束
- MVP 阶段由 1-2 人开发，开发周期控制在 8 周以内
- 第一期题库数量：基础层 ≥ 200 题，进阶层 ≥ 150 题，提高层 ≥ 100 题
- 服务器配置：初期使用 2 核 4G 云服务器 + 1 核 2G 数据库实例
- 不依赖第三方 AI/LLM 接口生成题目，所有出题由规则引擎完成

#### 4.5.4 设计约束
- 主色调以暖色系为主（橙、黄、浅绿），避免冷暗色调
- 字体大小：题目文字 ≥ 18px，按钮文字 ≥ 16px
- 不使用复杂手势操作（如长按、双击、多指手势）
- 不使用弹窗广告、强制分享、诱导评分等设计
- 确保每个页面都有明确的"返回"或"首页"入口，防止孩子迷路
