# 数字闯关 — 设计文档

**日期**: 2026-06-24  
**技术栈**: JavaFX + 本地 JSON  
**架构方案**: 单机桌面应用，Scene 切换 + 本地数据存档

---

## 一、总体架构

本项目采用 JavaFX 桌面应用架构，所有页面通过 Scene 切换完成。游戏状态统一保存在 `GameState` 中，存档使用本地 JSON 文件保存到用户目录。界面和业务逻辑在同一个桌面程序中完成，不依赖 Spring Boot 或 MySQL。

```text
TitleScene → MapScene → BattleScene → ResultScene
```

### 核心设计原则

- **单机优先**：不依赖外部后端服务，启动简单
- **本地存档**：使用 JSON 记录玩家进度
- **低龄友好**：大按钮、强反馈、清晰文字、像素风 UI
- **数据驱动**：题目、关卡、敌人数据尽量使用 JSON 配置

---

## 二、场景架构

### 2.1 Scene 切换

由 `SceneManager` 负责统一切换场景：
- `TITLE`：主菜单
- `MAP`：地图选关
- `BATTLE`：战斗
- `RESULT`：结算

`SceneFactory` 负责根据 `SceneType` 加载对应 FXML。

### 2.2 场景关系

- `TitleController`：开始新游戏、继续游戏
- `MapController`：关卡选择、难度切换
- `BattleController`：答题战斗、动画、倒计时
- `ResultController`：战斗结算、下一关、复活、返回地图

---

## 三、数据模型

### 3.1 GameState

`GameState` 是全局游戏状态容器，负责保存玩家当前进度。

主要字段：
- `playerName`：玩家名
- `hp / maxHp`：玩家血量
- `enemyHp / enemyMaxHp`：怪物血量
- `coins`：金币
- `level`：当前关卡
- `stars`：星级
- `round`：当前回合数
- `difficulty`：普通 / 困难
- `currentQuestion`：当前题目
- `options`：当前选项
- `countdown`：倒计时
- `battleFinished`：战斗是否结束
- `defeated`：是否失败复活状态
- `lastBattleVictory`：最近一次战斗是否胜利
- `lastBattleLevel`：最近一次战斗关卡
- `lastBattleRewardCoins`：最近一次奖励金币
- `lastBattleRewardStars`：最近一次奖励星级

### 3.2 Question

题目模型包含：
- `text`
- `answer`
- `options`

题目由 `QuestionGenerator` 根据关卡等级生成。

## 四、战斗系统设计

### 4.1 回合流程

```text
生成题目
  ↓
显示 4 个选项 + 倒计时
  ↓
玩家答题 / 逃跑
  ↓
答对 → 怪物扣血
答错 / 超时 → 玩家扣血
  ↓
判断胜负
  ↓
未结束则进入下一回合
```

### 4.2 倒计时规则

- 普通难度：10 秒
- 困难难度：3 秒
- 倒计时低于等于 3 秒时，UI 变红提示

### 4.3 HP 继承规则

- 每关开始时，玩家 HP **不重置**
- HP 会跨关卡继承
- 直到玩家 HP 归零时才进入失败流程
- 失败后复活会恢复满血

### 4.4 题目难度

`QuestionGenerator` 根据关卡等级生成题目：
- 低关卡：10 以内加减
- 中关卡：20 以内加减
- 更高关卡：100 以内加减
- 再往后：乘法、混合运算
- 题目类型覆盖整数加减乘除

### 4.5 动画与反馈

战斗页面包含以下反馈：
- 答对：玩家/怪物角色图切换 + 受击反馈
- 答错：玩家角色图受击
- 胜利：角色切换为胜利动作
- 超时失败：角色切换为失败动作 + 文字缩放淡出

## 五、地图系统设计

### 5.1 地图结构

地图页采用世界地图节点选择方式：
- 节点表示关卡
- BOSS 节点单独突出显示
- 玩家可切换普通 / 困难难度
- 入口包含返回标题按钮

### 5.2 关卡状态

后续可扩展为三种状态：
- 锁定
- 解锁
- 已完成

当前版本已具备节点式选择界面，后续可继续增加解锁逻辑和玩家定位标识。

## 六、结算系统设计

### 6.1 胜利结算

胜利时记录：
- 本次通关关卡
- 奖励金币
- 奖励星级

### 6.2 失败结算

失败时记录：
- 本次失败关卡
- 奖励为 0

### 6.3 结算动作

结算页支持：
- 下一关
- 复活
- 返回地图

---

## 七、存档系统设计

### 7.1 方案

当前采用 **本地 JSON 存档**，存档文件位于用户目录下：

```text
.quest-save.json
```

### 7.2 存档内容

存档包含：
- 玩家 HP / 金币 / 关卡 / 星级
- 难度设置
- 上次战斗结算信息

### 7.3 继续游戏

- 主菜单提供“继续游戏”按钮
- 若有存档则恢复进度
- 若无存档则不进入地图

## 九、界面风格设计

- 像素风复古 UI
- 大按钮、强对比配色
