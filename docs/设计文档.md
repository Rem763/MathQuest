## 4. 时序图

### 4.1 游戏启动时序

```text
App → SceneManager → SceneFactory → TitleController
App.start(Stage)
   ↓
SceneManager.start()
   ↓
switchTo(TITLE)
   ↓
SceneFactory.create(TITLE)
   ↓
TitleController.initialize()
   ↓
显示主界面
```

### 4.2 战斗答题时序

```text
用户 → BattleController → GameState → QuestionGenerator
点击选项
   ↓
BattleController.answerSelected(index)
   ↓
读取正确答案索引
   ↓
判断对错
   ├─ 正确 → 播放玩家攻击 / 怪物受击
   └─ 错误 → 播放怪物攻击 / 玩家受击
   ↓
resolveTurn()
   ↓
更新 HP / 金币 / 回合
   ↓
判断是否胜负结束
   ├─ 未结束 → 生成下一题
   └─ 结束 → 延迟切换到结果页
```

### 4.3 死亡结算时序

```text
HP 归零
   ↓
播放死亡动画
   ↓
PauseTransition 延迟
   ↓
finishBattle()
   ↓
SceneManager.switchTo(RESULT)
```

## 6. 数据模型设计

### 6.1 GameState

`GameState` 为运行期全局状态容器，保存当前游戏进度。

主要字段包括：
- `playerName`：玩家名称
- `hp / maxHp`：玩家当前血量与最大血量
- `enemyHp / enemyMaxHp`：怪物血量
- `coins`：金币
- `level`：当前关卡
- `stars`：当前星级
- `round`：战斗回合数
- `difficulty`：难度设置
- `currentQuestion`：当前题目文本
- `options`：题目选项
- `countdown`：倒计时
- `correctAnswerIndex`：正确答案索引
- `battleFinished`：战斗是否结束
- `defeated`：失败状态
- `lastBattleVictory`：最近一次战斗是否胜利
- `lastBattleLevel`：最近一次战斗关卡
- `lastBattleRewardCoins`：最近一次金币奖励
- `lastBattleRewardStars`：最近一次星级奖励
- `feedback`：战斗反馈文案

### 6.2 Question

`Question` 用于表示一道战斗题目，包含：
- `text`：题目文本
- `answer`：正确答案
- `options`：备选答案列表

### 6.3 SaveData

`SaveData` 为持久化存档结构，用于将 `GameState` 序列化至 JSON。

### 6.4 数据持久化

系统采用本地 JSON 文件作为存档介质，存档路径位于用户目录：

```text
.quest-save.json
```

## 7. 题目系统设计

### 7.1 题型范围

题目类型覆盖整数运算，包括：
- 加法
- 减法
- 乘法
- 除法

### 7.2 题目生成策略

`QuestionGenerator` 根据关卡等级生成题目，控制运算类型与数值范围：
- 低关卡：小范围加减
- 中关卡：更大范围加减
- 高关卡：乘法、混合运算
- 除法题保证整除，避免出现小数结果

### 7.3 选项生成

每道题生成 4 个选项，其中：
- 1 个正确答案
- 3 个干扰项

干扰项通过与正确答案做小幅偏移生成，以保证题目可判别且不易过于重复。

## 8. 战斗系统设计

### 8.1 回合流程

```text
生成题目 → 显示题目与选项 → 玩家作答/逃跑 → 判定对错 → 造成伤害 → 判断胜负 → 进入下一回合或结算
```

### 8.2 流程图

#### 8.2.1 单回合战斗流程

```text
开始回合
   ↓
生成题目
   ↓
显示题目与选项
   ↓
玩家答题 / 逃跑
   ↓
判定对错
   ↓
正确 → 玩家攻击怪物
错误 / 超时 → 怪物攻击玩家
   ↓
更新 HP / 金币 / 反馈
   ↓
判断胜负
   ├─ 未结束 → 下一回合
   └─ 结束 → 结果页
```

#### 8.2.2 失败结算流程

```text
玩家 HP 归零
   ↓
播放死亡动画
   ↓
延迟结束战斗
   ↓
进入结果页
```

#### 8.2.3 胜利结算流程

```text
怪物 HP 归零
   ↓
播放胜利/死亡反馈
   ↓
延迟结束战斗
   ↓
进入结果页
```

### 8.3 倒计时机制

- 普通难度：10 秒
- 困难难度：3 秒
- 当倒计时低于或等于 3 秒时，界面进行危险提示

### 8.4 伤害规则

- 答对：玩家对怪物造成固定伤害
- 答错或超时：怪物对玩家造成固定伤害
- 角色死亡后进入结算，不立即切换，需播放死亡动画后再结束

### 8.5 动画系统

战斗页支持：
- 玩家待机浮动
- 玩家攻击逐帧动画
- 玩家受击逐帧动画
- 玩家死亡逐帧动画
- 怪物待机浮动
- 怪物攻击逐帧动画
- 怪物受击逐帧动画
- 怪物死亡状态展示

动画资源采用如下目录结构：

```text
src/main/resources/sprites/player/
src/main/resources/sprites/slime/
```

### 8.6 战斗结算与延迟结束

- 玩家 HP 归零：先播放死亡动画，再延迟进入结果页
- 怪物 HP 归零：先播放胜利/死亡反馈，再延迟进入结果页

### 8.7 闯关记录展示

结果页会显示最近的闯关记录，记录内容包括：
- 关卡编号
- 胜负结果
- 回合数
- 奖励金币
- 奖励星级
- 完成时间

闯关记录在战斗结束时写入 `GameState`，并通过 `SaveData` 持久化到本地存档文件中，重新进入结果页时按最新记录顺序展示。

### 8.8 代码实现要点

战斗逻辑由 `BattleController` 完成，核心实现包括：
- `prepareBattle()`：进入战斗时重置状态并启动题目与倒计时
- `answerSelected()`：处理玩家选项点击
- `resolveTurn()`：统一处理伤害、胜负与下一回合
- `playPlayerAttackAnimation()`：播放玩家攻击与怪物受击
- `playMonsterAttackAnimation()`：播放怪物攻击与玩家受击
- `playTimeoutFailAnimation()`：播放失败反馈与死亡动画
- `finishBattle()`：停止动画并切换到结果页

### 8.8 动画资源映射

当前资源采用以下路径：

- 玩家待机：`/sprites/player/idle.png`
- 玩家攻击：`/sprites/player/attack/0.png` 起
- 玩家受击：`/sprites/player/hurt/0.png` 起
- 玩家死亡：`/sprites/player/dead/0.png` 起
- 怪物待机：`/sprites/slime/idle.png`
- 怪物攻击：`/sprites/slime/attack/0.png` 起
- 怪物受击：`/sprites/slime/hurt/0.png` 起
- 怪物死亡：`/sprites/slime/dead.png`

## 9. 地图系统设计

### 9.1 功能目标

地图页承担关卡导航与难度切换职责，用户在此完成战斗前的入口选择。

### 9.2 结构说明

- 节点表示关卡
- BOSS 节点单独突出显示
- 提供普通/困难模式切换
- 提供返回标题入口

### 9.3 状态管理

地图页读取 `GameState` 中的当前关卡、生命值、金币、难度等信息，并在界面上同步展示。

## 11. 异常处理设计

### 11.1 设计目标

异常处理目标是保证程序在资源缺失、存档异常、场景加载失败等情况下尽量不崩溃，并向用户提供可理解的降级反馈。

### 11.2 主要异常场景

- FXML 加载失败
- 图片资源缺失
- JSON 解析失败
- 存档文件不存在或损坏
- 场景切换过程中状态为空
- 题目数据异常或生成失败

### 11.3 处理策略

#### 资源加载异常
- 图片或 FXML 读取失败时，优先使用空处理或默认占位资源。
- 若关键资源缺失，记录错误并避免直接抛出到界面层。

#### 存档异常
- 如果后续启用存档恢复功能，继续游戏时若没有存档，返回标题页或提示用户重新开始。
- 存档损坏时忽略损坏数据并允许重新开局。

#### 题目生成异常
- 题目生成失败时使用兜底题目，避免战斗流程中断。
- 题目数据为空时返回安全默认题。

#### 场景与状态异常
- 切换场景前检查 `sceneManager` 和必要 UI 节点是否为空。
- 对战斗状态、选项数组等核心数据做空值保护。

### 11.4 用户反馈

- 轻量错误尽量在界面上显示简洁提示。
- 非致命异常不阻止用户继续游戏。
- 致命异常应阻止进入下一流程，但保留返回标题页的能力。

## 12. 非功能性设计

| 类别 | 设计要求 |
|------|----------|
| 性能 | 界面切换响应快速，战斗动画播放流畅 |
| 可用性 | 大按钮、明确反馈、适合低龄用户 |
| 可维护性 | 模块职责清晰，资源路径统一 |
| 可扩展性 | 题目、关卡、资源均可通过配置扩展 |
| 兼容性 | 面向 Windows 10/11 与 JDK 17+ 环境 |
