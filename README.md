## 1. 项目名称
**MathQuest — 小学生数学做题闯关系统**

MathQuest（数学探险），专为小学生打造的在线数学闯关练习平台。

## 2. 项目目标
为小学生设计一个简单、有趣、循序渐进的数学闯关做题项目，让孩子在游戏化体验中练习基础数学能力，提升计算兴趣和解题自信。

**核心目标拆解：**
- **学习目标**：覆盖小学 1-6 年级数学基础知识点，让学生在闯关中自然巩固计算能力
- **体验目标**：打造"像玩游戏一样做题"的体验，降低学生对数学练习的抵触心理
- **数据目标**：为家长和教师提供可视化的学习数据，让练习效果可量化、可追踪
- **技术目标**：构建一套可扩展的出题引擎和关卡系统，后续可快速扩展到其他学科

**成功指标（MVP 阶段）：**
- 学生平均每次练习完成率 ≥ 90%
- 单关平均用时控制在 3-8 分钟
- 错题重做率达到 60% 以上（说明学生愿意回头补弱）
- 家长端学习报告周活查看率 ≥ 70%

## 3. 目标用户

### 3.1 小学生（核心用户）
| 年级段 | 年龄 | 数学水平 | 核心需求 |
|--------|------|----------|----------|
| 1-2 年级 | 6-8 岁 | 初识加减法，数的概念建立中 | 题目简单、图形化、操作直观 |
| 3-4 年级 | 8-10 岁 | 掌握基本运算，开始接触乘除和应用题 | 适量挑战、看到进步、有成就感 |
| 5-6 年级 | 10-12 岁 | 综合运算能力形成，开始抽象思维 | 有难度区分、错题回顾、自我驱动 |

### 3.2 家长（辅助用户）
- 关注孩子学习习惯培养的家长（非"鸡娃"型，更看重兴趣引导）
- 需要客观数据了解孩子薄弱环节，而非仅靠主观判断
- 有控制孩子屏幕时间的需求，希望设置使用时长和题量上限

### 3.3 教师（扩展用户，MVP 阶段做简化版）
- 小学数学教师，需要快速生成课堂小练或课后巩固题
- 关注班级整体正确率分布，快速定位共性薄弱知识点
- 操作路径要极短：选年级 → 选知识点 → 生成 → 使用，三步完成

## 5. 核心玩法

### 5.1 闯关模式
游戏化闯关是本项目的核心体验骨架。整个系统由 **章节 → 关卡** 两级结构组成：

- 玩家从第 1 关开始，必须按顺序挑战，不可跳过
- 每个关卡包含 5-10 道数学题（低年级 5-6 题，高年级 8-10 题）
- 正确率达到该关阈值（默认 80%）即通关，自动解锁下一关
- 未达标可无限重试，不设惩罚
- 通关后显示星级评价（1-3 星）和获得的奖励

**为什么用线性闯关而非自由选择？**
- 避免选择焦虑：小学生面对太多选项会不知所措
- 保证难度递进：防止学生跳入过难的关卡产生挫败感
- 制造期待感：锁住的关卡天然激发好奇心和解锁欲望

### 5.2 题目类型
以基础数学题为主，六大题型覆盖不同练习目标：

| 题型 | 说明 | 适用知识点 | 示例 |
|------|------|-----------|------|
| 口算题 | 直接输入答案，最核心题型 | 加减乘除、混合运算 | 23 + 15 = ? |
| 选择题 | 4 选 1，降低输入门槛 | 概念判断、比较大小 | 哪个数最大？A.18 B.23 C.9 D.31 |
| 填空题 | 填写缺失数字，训练逆向思维 | 数的分解、凑十法 | 8 + ___ = 15 |
| 判断题 | 对/错二选一，巩固概念 | 运算性质、等式判断 | 3+5 = 5+3 对吗？ |
| 连线题 | 拖拽连线，适合低年级 | 数物对应、口诀配对 | 将算式与答案连线 |
| 应用题 | 图文结合的生活场景题 | 综合运用、理解能力 | 小明有12颗糖... |

**MVP 阶段优先实现：** 口算题、选择题、填空题。连线题和应用题在第二阶段加入。

### 5.3 题库规则设计

#### 5.3.1 题库分层
题库按照难度和知识点进行分层管理：

| 层级 | 知识点范围 | 数字范围 | 题量要求（MVP） |
|------|-----------|---------|----------------|
| **基础层** | 10 以内加减法、数的认识与比较 | 0-10 | ≥ 200 题 |
| **进阶层** | 20 以内加减法（含进位退位）、100 以内整十加减 | 0-100 | ≥ 150 题 |
| **提高层** | 乘法口诀（1-9）、混合运算（两步）、简单应用题 | 0-100 | ≥ 100 题 |

#### 5.3.2 出题规则（详细）
- **单知识点原则**：每道题只考察一个知识点，不出现"进位加法 + 乘法"这种复合考点
- **题型一致性**：同一关卡内题型不超过 2 种，保证学生精力集中
- **数字范围控制**：关卡 N 的数字最大值 = 关卡 N-1 的最大值 × 1.2（逐步扩大）
- **生活化优先**：优先使用"分苹果、买文具、排队做操"等校园/家庭场景
- **禁止项**：超纲题、歧义题、文字超过 30 字的题（低年级）、涉及金钱数额过大的题

#### 5.3.3 题目生成方式（三种模式混合）

```
出题引擎
├── 固定题库模式（40%）
│   └── 适用于：基础题、高频考点、需要精心设计的应用题
│   └── 特点：每道题经过人工审核，质量最高
├── 规则生成模式（50%）
│   └── 适用于：加减法、乘法口诀等规律性强的题目
│   └── 特点：系统根据参数（运算符、数字范围）自动生成，题库无穷
└── 混合模式（10%）
    └── 适用于：需要一定随机性但又要控制质量的场景
    └── 特点：固定模板 + 随机数字填充（如"小明有{X}个苹果，吃了{Y}个，还剩几个？"）
```

#### 5.3.4 题目校验规则（出题引擎必检项）
- [ ] 答案唯一性检查：每道题有且仅有一个正确答案
- [ ] 干扰项合理性：选择题的干扰项 = 正确答案 ± 常见错误偏移量
- [ ] 非负检查：运算结果不可为负数（除法不可有余数，低年级）
- [ ] 数字范围检查：所有数字在关卡允许范围内
- [ ] 文字审核：题干不含超纲词汇、不涉及敏感内容
- [ ] 去重检查：同一关卡内不出现数字不同但结构完全相同的题

#### 5.3.5 难度递进规则（五步递进法）
1. **第一步**：先练单步题（如单纯加法），不引入混合
2. **第二步**：再扩大数字范围（10→20→50→100），保持单步
3. **第三步**：加入口算变式（填空题、判断题），丰富题型
4. **第四步**：引入两步运算（先乘后加、先减后加），增加步骤
5. **第五步**：加入文字应用题，训练读题和转化能力

每一关正确率 ≥ 80% 才开放本难度的下一关；连续三关正确率 ≥ 90% 则自动跃迁到更高难度。

#### 5.3.6 错题处理规则（闭环流程）
```
答错 → 即时显示正确答案和简短提示
     → 自动加入错题本（记录错题时间、知识点、用户答案）
     → 错题在后续同类关卡中以 30% 概率再次出现
     → 同一知识点连续错 3 题 → 系统推荐该知识点专项练习包
     → 错题重做正确 2 次 → 从活跃错题库移除（历史保留）
```

### 5.4 关卡设计

#### 5.4.1 关卡结构
```
章节（Chapter）
├── 关卡 1-1（入门）
├── 关卡 1-2（进阶）
├── 关卡 1-3（挑战）
├── 关卡 1-4（综合）
└── 关卡 1-5（Boss关，限时模式）
```

每个章节 4-5 个小关卡，前 3-4 关为常规练习，最后一关为 Boss 关（综合 + 限时）。

#### 5.4.2 关卡划分（六大阶段，对应六个章节）

| 阶段 | 章节名 | 关卡数 | 年级定位 | MVP 优先级 |
|------|--------|--------|----------|-----------|
| 第 1 阶段 | 数的启蒙 — 10 以内加减法 | 5 关 | 一年级上 | P0（必须） |
| 第 2 阶段 | 进位初探 — 20 以内加减法 | 5 关 | 一年级下 | P0（必须） |
| 第 3 阶段 | 大数挑战 — 100 以内加减法 | 5 关 | 二年级 | P0（必须） |
| 第 4 阶段 | 口诀达人 — 乘法口诀 | 5 关 | 二年级 | P1（MVP 含） |
| 第 5 阶段 | 混合运算 — 两步计算 | 5 关 | 三年级 | P2（第二期） |
| 第 6 阶段 | 生活数学 — 简单应用题 | 5 关 | 三-四年级 | P2（第二期） |

#### 5.4.3 关卡解锁规则（详细）
- 新用户默认仅第 1 阶段第 1 关解锁，其余全部锁定
- 当前关正确率 ≥ 80% → 自动解锁同一章节的下一关
- 当前章节所有关卡完成 → 自动解锁下一章节第 1 关
- 未通过的关卡显示"重试"按钮，可无限次挑战
- 已通关的关卡显示星级，点击可重新挑战（刷星）
- **跨级解锁特例**：连续三个关卡获得三星 → 可跳过本章节剩余关卡，直接进入下一章节

#### 5.4.4 关卡目标规则
| 评价等级 | 正确率要求 | 时间要求 | 奖励 |
|----------|-----------|---------|------|
| ⭐ 一星 | ≥ 60% | 无要求 | 基础积分 + 解锁下一关 |
| ⭐⭐ 二星 | ≥ 80% | 无要求 | 1.5 倍积分 + 下一关解锁 |
| ⭐⭐⭐ 三星 | ≥ 90% | 在目标时间内 | 2 倍积分 + 特殊徽章 |

每个关卡都设有"建议完成时间"，用来帮助学生建立时间观念，但不做强制要求（除 Boss 关外）。

### 5.5 题型设计

#### 5.5.1 题型分类与交互设计

| 题型 | 交互方式 | 适用年级 | 得分权重 |
|------|---------|---------|---------|
| **口算题** | 数字键盘输入答案，回车提交 | 全年级 | ×1.0 |
| **选择题** | 点击选项直接提交，选后即判 | 1-4 年级为主 | ×0.9 |
| **填空题** | 点击空白处弹出数字键盘 | 2-6 年级 | ×1.1 |
| **判断题** | 点击"对"或"错"按钮 | 1-3 年级 | ×0.8 |
| **连线题** | 拖拽左侧算式到右侧答案 | 1-2 年级 | ×1.2 |
| **应用题** | 阅读题目 → 输入算式 → 输入答案 | 3-6 年级 | ×1.5 |

#### 5.5.2 各题型使用场景
- **口算题**（使用频率 ~40%）：核心题型，用于强化纯计算能力，适合每日练习
- **选择题**（~25%）：降低输入门槛，适合概念理解和低年级初学阶段
- **填空题**（~15%）：训练逆向思维和数字感，适合中高年级
- **判断题**（~10%）：快速巩固概念，适合课堂随练
- **连线题/应用题**（~10%）：丰富体验，适合阶段性综合练习

#### 5.5.3 题型配置原则
- 1-2 年级关卡：口算题 40% + 选择题 40% + 判断题 20%
- 3-4 年级关卡：口算题 40% + 填空题 30% + 选择题 20% + 应用题 10%
- 5-6 年级关卡：口算题 30% + 填空题 25% + 应用题 25% + 选择题 20%
- 同一关卡内题型的切换频率：每 2-3 题切换一次（不要太频繁）

### 5.6 评分规则

#### 5.6.1 基础评分（单题得分）
```
单题得分 = 基础分(10分) × 题型系数
         + 速度奖励(0-5分，根据答题速度)
         + 连击奖励(连对3题起，额外+2分/题)
```

- 基础分：每道题 10 分
- 速度奖励：在"标准时间"的 50% 内完成 → +5 分；100% 内完成 → +2 分；超过 → +0 分
- 连击奖励：连续答对 3 题触发，之后每多对 1 题额外 +2 分（连击中断重置）

#### 5.6.2 通关评分（整关结算）
```
通关总分 = Σ单题得分
通关正确率 = 答对题数 / 总题数 × 100%
通关星级 = 根据正确率+时间综合判定
```

#### 5.6.3 星级评价标准（具体数值）
| 星级 | 正确率 | 额外条件 |
|------|--------|---------|
| ⭐ 一星 | 60%-79% | 完成全部题目即可 |
| ⭐⭐ 二星 | 80%-94% | 完成全部题目 |
| ⭐⭐⭐ 三星 | 95%-100% | 总用时 ≤ 标准总时间的 120% |

#### 5.6.4 扣分与容错机制
- **不设扣分**：答错不扣基础分，只失去该题的速度奖励和中断连击
- **二次机会**：Boss 关以外的关卡，答错后可点击"再试一次"（消耗该题 50% 得分，答对给一半分）
- **保底机制**：同一关重试 3 次仍未通过 → 系统自动降低该关难度 1 档（题目不变，降低通关阈值至 60%）
- **评分导向**：始终鼓励"完成"和"进步"，不鼓励"排名"和"比较"

### 5.7 激励机制（四层激励体系）

```
第一层：即时反馈层
├── 单题答对 → 星星+1 动画 + 音效
├── 连击触发 → 火焰特效 + "太棒了！"文字
└── 答错 → 柔和提示音 + "再想想哦~"（不显示红色叉）

第二层：关卡结算层
├── 通关 → 通关面板（得分、正确率、星级动画）
├── 三星 → 金色徽章 + 特殊音效
└── 解锁新关 → "新关卡已开启！"动画

第三层：成就系统层
├── 里程碑徽章：完成10关、50关、100关...
├── 专项徽章：全连(连对10题)、速算王(平均速度<3秒)、
│              全勤王(连续7天练习)、逆袭王(错题重做正确率>90%)
└── 章节徽章：每通过一个大章节获得一枚主题徽章

第四层：社交激励层（第二期）
├── 好友排行榜（仅显示排名，不显示具体分数，避免攀比）
└── 班级挑战赛（教师发起，限时集体闯关）
```

## 6. 难度设计

### 6.1 难度分级（四级体系）

| 难度级别 | 数字范围 | 运算类型 | 题目示例 |
|----------|---------|---------|---------|
| **入门级** | 0-10 | 加法、减法 | 3 + 5 = ? |
| **基础级** | 0-20 | 加减法（含进退位）、数的比较 | 15 - 8 = ? |
| **进阶级** | 0-100 | 加减法、乘法口诀、简单除法 | 7 × 8 = ? |
| **提高级** | 0-1000 | 混合运算、应用题 | (23 + 15) × 2 = ? |

### 6.2 难度递进原则
- **一次只变一个变量**：扩大数字范围时保持运算类型不变；引入新运算时缩小数字范围
- **难度增量 ≤ 15%**：相邻关卡的难度差异控制在 15% 以内（以正确率变化衡量）
- **题目先短后长**：低年级题干 ≤ 15 字，中年级 ≤ 30 字，高年级 ≤ 60 字
- **提示梯度**：
  - 第 1 次答错：显示"再想想"
  - 同题第 2 次答错：显示"提示：可以用凑十法试试"
  - 同题第 3 次答错：显示完整解题步骤，让学生看懂后再进入下一题

### 6.3 自适应难度（第二期功能设计预留）
- 连续 5 题答对 → 临时提升难度半档
- 连续 3 题答错 → 临时降低难度半档
- 自适应范围不超过学生当前年级 ±1 级的范围
- 每个关卡结束时记录"适应后难度"，作为下次推荐起始难度

## 7. 用户流程

### 7.1 学生主流程（核心路径）
```
首页（显示当前进度、推荐关卡、今日任务）
  │
  ├─→ [开始闯关] → 关卡选择页（章节+关卡列表，已解锁/未解锁/已完成）
  │       │
  │       └─→ 答题页（逐题展示，答一题判一题）
  │              │
  │              ├─→ 答对 → 星星动画 → 自动进入下一题
  │              └─→ 答错 → 柔提示 → 显示答案 → 自动进入下一题
  │              │
  │              └─→ 最后一题完成 → 通关结果页
  │                     │
  │                     ├─→ [再来一次] → 重新挑战本关
  │                     ├─→ [下一关] → 进入下一关卡（刚解锁的）
  │                     └─→ [返回首页]
  │
  ├─→ [错题本] → 按知识点分类的错题列表
  │       │
  │       └─→ 点击某道错题 → 显示解析 → [重新练习]
  │              └─→ 进入针对该知识点的专项练习
  │
  └─→ [学习报告] → 本周/本月统计图表
```

### 7.2 家长流程
```
进入家长入口（密码/生物识别验证）
  │
  ├─→ [学习概览] → 孩子本周练习天数、总题数、正确率
  ├─→ [详细报告] → 正确率趋势图、各知识点掌握雷达图、常错题型排行
  ├─→ [错题总览] → 查看所有未消化的错题，按时间/知识点筛选
  └─→ [设置中心] → 年级选择、每日题量上限、练习时段、音效开关、
                     护眼模式、修改密码
```

### 7.3 教师流程（MVP 阶段做简化版）
```
选择年级/知识点 → 预览自动生成的题目列表
  │
  ├─→ 手动增删替换题目
  ├─→ 生成练习题单（可打印版本）
  └─→ 查看班级整体完成统计（如有学生账号关联）
```

### 7.4 首次体验流程（游客模式）
```
打开首页 → 无需注册即可试做 3 道题（引导体验关）
  │
  ├─→ 完成引导关 → 弹出"注册解锁更多关卡"提示
  │       ├─→ [立即注册] → 注册页 → 选择角色 → 完成注册 → 进入首页
  │       └─→ [稍后再说] → 返回首页（只能重复玩体验关）
  │
  └─→ 注册后首次登录 → 新手指引动画（手把手引导完成第一关）
```

**体验关设计要点：**
- 仅包含 3 道最简单的 10 以内加法题，确保孩子能轻松完成
- 体验关中不显示任何设置或复杂选项
- 体验数据不保存（除非注册）
- 引导动画要短（≤ 30 秒），不强制、可跳过

### 7.5 密码找回流程
```
登录页 → 点击"忘记密码"
  │
  ├─→ 输入注册邮箱/手机号
  ├─→ 系统发送重置链接（有效期 30 分钟）
  ├─→ 点击链接 → 输入新密码 → 完成重置
  └─→ 若家长设置了备用联系方式 → 同时发送至家长端确认
```

**安全措施：**
- 重置链接一次性使用
- 同账号 1 小时内最多申请 3 次重置
- 新密码长度 ≥ 8 位，须含字母和数字
- 重置成功后所有已登录设备强制下线

### 7.6 多人共用设备场景
```
设备首页 → 显示已有账号头像列表（最多 4 个）
  │
  ├─→ 点击自己的头像 → 输入密码 → 进入个人首页
  ├─→ 点击"+添加" → 注册新账号（需要家长密码验证）
  └─→ 游客入口 → 体验关（无需登录）
```

**设计要点：**
- 同一设备记住最近 4 个登录账号
- 每个账号的学习进度完全隔离
- 切换账号时当前答题进度自动保存
- 家长账号可查看该设备上所有关联的学生账号

## 8. 页面设计

### 8.1 首页
**功能定位**：学生登录后的第一屏，承担"导航 + 激励 + 快速开始"三重功能

**页面布局（从上到下）：**
1. **顶部状态栏**：头像 + 昵称 + 当前等级/称号 + 星星总数/金币数
2. **进度卡片**：当前章节进度条（如"第2章：进位初探 — 已完成 3/5 关"）
3. **今日任务**：每日 3 个小目标（如"完成任意 1 关""答对 15 题""重做 5 道错题"），完成打勾
4. **快捷入口区**（3 个大按钮）：
   - 🎮 继续闯关（跳转到上次退出的关卡）
   - 📖 错题本（带角标数字显示未消化的错题数）
   - 📊 学习报告
5. **底部导航栏**：首页 / 闯关 / 错题本 / 报告 / 我的

### 8.2 关卡选择页
**功能定位**：展示完整的章节关卡地图，让学生一目了然"走到哪了"

**页面设计要点：**
- 使用"地图路径"视觉风格，关卡节点用圆形图标表示
- 已通关：⭐ 标记 + 彩色填充
- 当前可挑战：发光/跳动动画引导点击
- 未解锁：灰色 + 小锁图标，点击显示解锁条件提示
- 点击当前关卡节点 → 弹出关卡详情卡片（关卡名、题数、目标正确率、建议时间）→ 点击"开始挑战"
- 点击已通关关卡 → 弹出历史成绩（得分、用时、星级）→ 可[重新挑战]

### 8.3 答题页
**功能定位**：核心做题页面，需要极简、清晰、无干扰

**页面布局：**
1. **顶部信息栏**：关卡名 + 当前题号/总题数（如"第 3/8 题"）+ 得分 + 连击状态
2. **题目展示区**（占页面 50%）：大字号显示题目，应用题配插图
3. **答题交互区**（占页面 40%）：
   - 口算题：数字键盘 + 输入框 + 提交按钮
   - 选择题：4 个大按钮（A/B/C/D），横向 2×2 排列
   - 填空题：数字键盘 + 高亮填空位
   - 判断题：两个大按钮（✓ 对 / ✗ 错）
4. **辅助按钮**：提示（消耗提示次数）、跳过（不记录分数）
5. **答题反馈**：提交后答题区域短暂替换为对/错反馈动画（0.8 秒），自动进入下一题

**交互细节：**
- 数字键盘按键 ≥ 56×56px（适合手指点击）
- 选项按钮间距 ≥ 16px（防止误触）
- 关闭浏览器/返回时弹出"确定退出吗？进度将丢失"二次确认
- 支持键盘快捷键：数字键输入、回车提交、方向键切换选项

### 8.4 通关结果页
**功能定位**：关卡完成的结算反馈，强化成就感

**页面内容从上到下：**
1. **星级揭晓**（动画）：1-3 颗星依次点亮，3 星时触发满星特效
2. **成绩面板**：
   - 总得分（大号字体）
   - 正确率：X/X（100%）
   - 用时：X 分 X 秒
   - 连击：最高 X 连击
3. **成就展示**（如有新获得）：新徽章弹出动画
4. **奖励展示**：获得星星数、金币数、是否解锁新关卡
5. **操作按钮**：
   - [下一关 ▶]（主按钮，突出显示，刚刚解锁的新关）
   - [再来一次 🔄]（重新挑战本关，刷更高星级）
   - [返回首页 🏠]

### 8.5 错题本页
**功能定位**：错题管理中心，帮助学生有针对性地补弱

**页面结构：**
1. **筛选栏**：按知识点筛选、按时间范围筛选（本周/本月/全部）
2. **错题列表**：每行显示题目摘要 + 错误次数 + 最近错误时间
3. **统计数据条**：顶部显示"未消化错题：X 道 / 历史错题：Y 道"
4. **操作**：
   - 点击单个错题 → 展开显示完整题目 + 正确答案 + 解题思路
   - [练习此题] → 直接进入单题练习模式
   - [同类题练习] → 系统生成同知识点的 5 道题
   - [批量清空] → 将所有已纠正的错题标记为"已消化"（需二次确认）

### 8.6 学习报告页
**功能定位**：用图表直观展示学习趋势，服务于家长和学生自我复盘

**页面内容（MVP 版）：**
1. **概览卡片**（横向 4 个数字卡片）：
   - 累计练习天数
   - 累计答题数
   - 当前正确率
   - 已获徽章数
2. **正确率趋势折线图**：近 7 天/近 30 天的每日正确率变化
3. **知识点掌握雷达图**：各知识点的正确率分布（如加法 95%、减法 88%、乘法 72%）
4. **常错题型排行**：Top 5 易错知识点，可点击跳转到对应的错题练习

### 8.7 家长设置页
**功能定位**：家长控制中心，管理孩子的使用方式和范围

**设置项列表：**
| 设置项 | 类型 | 说明 |
|--------|------|------|
| 年级选择 | 下拉框 | 1-6 年级，影响默认题目难度 |
| 每日题量上限 | 滑块 | 10-100 题/天，默认 50 题 |
| 每日时长上限 | 滑块 | 15-90 分钟/天，默认 45 分钟 |
| 可使用时段 | 时间选择 | 如"仅限 18:00-20:00"，其余时间不可用 |
| 音效 | 开关 | 开启/关闭背景音乐和音效 |
| 护眼模式 | 开关 | 暖色温 + 每 20 分钟休息提醒 |
| 修改密码 | 按钮 | 修改家长入口密码 |

### 8.8 "我的"页面
**功能定位**：个人中心和系统设置的聚合入口

**页面内容：**
1. **个人信息区**：头像、昵称、当前等级/称号、年级
2. **成就展示区**：已获徽章墙（网格展示，未获得的灰色剪影）
3. **数据面板**：累计练习天数、总答题数、总正确率、最高连击
4. **功能入口**：
   - 学习报告（跳转至报告页）
   - 错题本（带未消化数量角标）
   - 家长入口（需密码验证，学生不可见）
5. **系统设置**：
   - 音效开关
   - 背景音乐开关
   - 护眼模式
   - 夜间模式（深色主题）
6. **账号管理**：
   - 修改昵称
   - 修改密码
   - 切换账号（多人共用设备场景）
   - 退出登录

## 11. 内容设计原则
- **语言要求**：使用小学生熟悉的语言，题干中不出现超出对应年级语文课本生字表的词汇
- **场景选择**：应用题场景贴近校园、家庭、购物、运动、游戏等日常环境
- **题量控制**：每次练习 5-10 题，确保在 3-8 分钟内完成，匹配小学生的注意力跨度
- **风格定位**：题目风格轻松、正面、鼓励性，避免"你错了""不及格"等负面表达
- **趣味与教学平衡**：趣味性是手段，教学效果是目的，不为好玩而牺牲题目质量
- **错误处理**：任何非正常操作（断网、误关闭、输入异常）都不应导致数据丢失或体验中断

## 14. 项目价值
- **对学生**：在轻松的游戏化环境中练习数学，把"做题"从负担变成挑战和乐趣
- **对家长**：用数据了解孩子的学习情况，从"盲目督促"变成"精准辅助"
- **对教师**：降低出题和批改的重复劳动，把精力集中在教学本身
- **对开发团队**：构建一套可复用的出题引擎和关卡框架，为后续扩展到语文、英语等学科打下基础

## 15. 项目结论与下一步
这是一个适合小学生的数学闯关练习项目，核心理念是**"简单、好玩、可持续练习"**。

**近期行动项：**
1. 确认 MVP 阶段的 4 个章节 30 个关卡的具体参数（每题数字范围、题型配比、通关阈值）
2. 完成出题引擎的原型代码，验证规则生成的可行性和题目质量
3. 准备基础题库的第一批 200 道人工审核题目
4. 搭建前后端项目骨架，调通第一个接口

建议从第 1 阶段（10 以内加减法）开始，先做最小可用版本完成端到端流程，验证核心体验后再逐步丰富后续阶段。

## 16. 风险与应对

| 风险分类 | 风险描述 | 影响程度 | 发生概率 | 应对措施 |
|----------|---------|---------|---------|---------|
| **产品风险** | 题目太难或太简单，孩子失去兴趣 | 高 | 中 | 第7周邀请5+名小学生试做，根据数据校准难度；长期做自适应难度 |
| **产品风险** | 家长不认可"游戏化"方式，觉得不严肃 | 中 | 低 | 家长端突出学习数据和进步可视化，弱化游戏元素的可配置性 |
| **技术风险** | 出题引擎生成的题目存在大量重复 | 高 | 中 | 第2周优先完成出题引擎原型并测试，确保去重算法有效 |
| **技术风险** | 并发答题时系统响应变慢 | 中 | 中 | 题目预生成 + Redis 缓存，单服务器目标支撑 500 并发 |
| **合规风险** | 收集未成年人信息涉及隐私合规 | 高 | 中 | 仅收集必要信息（昵称+年级+密码），不收集真实姓名、学校、定位等 |
| **合规风险** | 未经监护人同意收集儿童数据 | 高 | 低 | 注册时增加家长验证步骤（邮箱/手机确认），家长设置独立的控制密码 |
| **资源风险** | 1-2人开发8周不够完成全部MVP | 中 | 高 | 严格按 P0→P1 优先级开发，每周围绕"可演示的最小闭环"交付 |
| **运营风险** | 题库初期数量不足导致题目重复率高 | 中 | 中 | 规则生成题占50%保证题量，固定题库逐步积累至450+题 |

**风险应对总原则：**
- MVP 阶段优先化解"孩子不喜欢"和"题目重复"两大核心风险
- 合规风险在开发前咨询法律意见，确保符合《个人信息保护法》和《未成年人保护法》
- 技术风险通过早期原型验证出题引擎可行性

## 17. 通知机制

| 通知类型 | 触发条件 | 触达方式 | 发送频率 | MVP 阶段 |
|----------|---------|---------|---------|---------|
| 学习周报 | 每周日自动生成 | 站内消息（首页角标提醒） | 每周 1 次 | P1 |
| 连续打卡提醒 | 连续 2 天未练习 | 站内消息 | 每天 1 次 | P1 |
| 成就获得通知 | 获得新徽章/里程碑 | 即时弹窗 + 站内消息 | 即时 | P1 |
| 错题预警 | 同一知识点连续错 3 题 | 站内消息 + 首页推荐练习 | 条件触发 | P1 |
| 休息提醒 | 连续练习超 20 分钟 | 页面内提示弹窗 | 每 20 分钟 | P1 |

## 18. 题目版权与来源

| 来源 | 占比 | 说明 |
|------|------|------|
| **规则生成** | 50% | 系统根据运算规则自动生成，无版权问题 |
| **原创题库** | 40% | 项目团队自主编写，版权归项目所有 |
| **开放资源改编** | 10% | 参考公开教材和习题集改编，不直接复制 |

**版权合规要点：**
- 固定题库的每道题由人工编写，确保不直接复制市面教材
- 应用题场景原创设计，不使用现有品牌、商标、商业 IP
- 如后续引用《义务教育数学课程标准》知识点体系，仅引用知识点结构（不受版权保护），不复制原文表述
- 题库数据作为项目核心资产，需单独备份和版本管理

## 19. 术语表

| 术语 | 全称/说明 |
|------|----------|
| **MVP** | Minimum Viable Product，最小可用版本 |
| **P0/P1/P2/P3** | 优先级分级：P0 必须，P1 推荐，P2 二期，P3 远期 |
| **关卡** | Level，一次完整的练习单元，包含 5-10 道题 |
| **章节** | Chapter，一组相关关卡的集合，对应一个知识点阶段 |
| **连击** | Combo，连续答对多道题触发的奖励状态 |
| **星级** | 1-3 星的关卡评价，综合正确率+时间判定 |
| **错题本** | 自动收录答错题目的虚拟笔记本 |
| **消化** | 错题重做正确 2 次以上，标记为已掌握 |
| **Boss 关** | 每个章节最后的综合关卡，包含限时挑战 |
| **出题引擎** | Question Generator，自动根据关卡参数生成题目的核心模块 |
| **会话** | Session，一次完整的闯关流程（从开始到结算） |
| **知识点** | Knowledge Point，一个独立的数学知识单元 |
| **自适应难度** | 根据学生答题表现动态调整题目难度的机制 |
| **JWT** | JSON Web Token，无状态身份认证方案 |
| **OCR** | 光学字符识别，后续可能用于手写答案识别（远期规划） |
